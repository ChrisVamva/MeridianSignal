# Agent Framework Selection

## Selection Criteria

The archive repeatedly compares frameworks by a few stable dimensions.

Those dimensions deserve preservation:

- provider flexibility
- tool support
- orchestration depth
- memory handling
- ease of local or self-hosted deployment
- cost discipline
- fit for the actual workflow

## Durable Preference

Prefer agent frameworks that keep the model layer swappable.

Why:

- free tiers change
- pricing changes
- performance shifts
- privacy requirements vary
- different tasks benefit from different model strengths

## Strong Positive Signal

Frameworks that are model-agnostic or built around broad provider support are strategically safer than systems tightly coupled to one vendor.

## Caution

A framework can look powerful on paper but still be the wrong choice if:

- setup cost is too high
- the workflow is simpler than the framework assumes
- the system encourages unnecessary complexity

## Working Rule

Choose the lightest framework that still supports the real loops, tools, and memory requirements of the task.
