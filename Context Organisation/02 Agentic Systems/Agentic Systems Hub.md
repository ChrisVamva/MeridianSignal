# Agentic Systems Hub

## Definition

Agentic systems are systems that do more than answer.
They observe, choose, execute, check, and sometimes continue across sessions or tools.

## Why This Area Is Central

The archive returns constantly to agents:

- terminal agents
- coding agents
- research agents
- custom role agents
- OpenClaw
- multi-agent orchestration
- memory and session design

This is not incidental.
It is one of the main engines of the whole archive.

- When is a prompt enough?
- When does a workflow require an agent loop?
- How much memory should an agent carry?
- Which parts of autonomy are useful and which are dangerous?
- Which frameworks are flexible enough to avoid lock-in?

## Durable Stance

The archive favors agents when:

- the task is recurrent
- the task contains loops
- the task needs tool use
- the task benefits from persistent structure or memory

The archive should resist agents when:

- the task is one-shot and simple
- the system cost exceeds the real value
- the autonomy introduces unnecessary risk
