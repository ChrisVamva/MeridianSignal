# Tool Evaluation Criteria

## Primary Test

A tool is valuable when it improves capability without creating disproportionate complexity.

## Core Criteria

### Strategic Fit

Does it support the actual research pattern or only an imagined one?

### Operational Usefulness

Can it be deployed, integrated, or reused in a concrete workflow?

### Replaceability

Can the wider system survive if the tool disappears, degrades, or becomes expensive?

### Cost Discipline

Does the tool justify its cost in money, time, tokens, or attention?

### Legibility

Can its role be explained clearly inside the archive?
If not, it may be creating fog instead of leverage.

### Security And Control

Does it introduce permissions, data exposure, or dependency risks?

## Signals Of Weak Tool Notes

- excitement without criteria
- comparison without conclusion
- repetition of marketing claims
- no link to a real use case

## Signals Of Strong Tool Notes

- explicit reasoning
- operational consequence
- tradeoff awareness
- clear fit or non-fit with the larger system
