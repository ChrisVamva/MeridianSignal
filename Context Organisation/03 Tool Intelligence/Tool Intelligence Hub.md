# Tool Intelligence Hub

## Definition

Tool intelligence is not a pile of product notes.
It is the disciplined evaluation of which tools actually deserve a place in the research and execution stack.

## Why This Area Needs Heavy Curation

The original archive contains a huge amount of tool material, but not all of it deserves equal weight.

It includes:

- useful audits
- deployable configurations
- strategic comparisons
- repeated ecosystem snapshots
- hype-driven accumulation

This new layer keeps the first three and compresses the rest.

## Stable Evaluation Questions

- Does the tool solve a real recurring problem?
- Is it compatible with the wider operating system?
- Does it increase leverage or merely add complexity?
- Is it replaceable if the provider changes terms?
- Does it create security or dependency risks?
