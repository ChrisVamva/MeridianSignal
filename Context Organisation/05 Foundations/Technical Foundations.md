# Technical Foundations

## Role

Technical foundations matter here because they support higher-order work.
They are not preserved as a curriculum for its own sake.

## Core Concepts Worth Keeping Clean

### API

An API is the contract that lets one software component talk to another.
This matters because the archive is full of systems that only become useful when they can call tools, models, or services programmatically.

### JSON

JSON matters because many of the archive's systems exchange structured information through it.

### Git

Git matters because durable work requires versioned change rather than accidental drift.

### Node.js and Python

These matter not as abstract languages, but as the main implementation substrates surrounding many tools and projects in the archive.

### MCP and Tool Interfaces

The archive repeatedly touches protocols and integration layers.
Those belong in foundations when they are written clearly enough to support future building.

## Preservation Rule

Foundational notes should remain short, plain, and linked upward to larger architectures.
