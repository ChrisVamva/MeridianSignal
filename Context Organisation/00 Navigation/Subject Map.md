# Subject Map

## Core Shape

The archive revolves around one central ambition:

Build a durable research-and-action system in which tools, agents, projects, and conceptual frameworks reinforce one another.

## Subject Relationships

### Investigation Operating System

This is the method layer.
It defines how research is gathered, evaluated, routed, decomposed, and converted into action.

### Agentic Systems

This is the execution layer.
It covers digital workers, agent roles, orchestration, memory, and autonomy boundaries.

### Tool Intelligence

This is the selection layer.
It evaluates which platforms, runtimes, IDEs, frameworks, and model stacks are worth attention.

### Projects and Experiments

This is the proof layer.
It records where ideas were tested in concrete work.

### Foundations

This is the support layer.
It contains technical primitives and clean conceptual basics.

### Knowledge Architecture

This is the synthesis layer.
It handles semantics, symbolic systems, structured insight extraction, and worldview-level integration.

### Strategic Filters

This is the judgment layer.
It records failure modes, anti-patterns, and caution rules.

## Long-Term Logic

The intended cycle is:

1. Study a capability.
2. Evaluate tools and agent patterns around it.
3. Integrate the useful parts into the investigation operating system.
4. Test ideas through projects.
5. Distill lessons into durable conceptual notes.
6. Sharpen strategic filters so future work improves.
