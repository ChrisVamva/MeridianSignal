# Research Terrain Map

## Purpose

This note defines the durable territory the research keeps returning to.

It is the bridge between the long-term subject structure of `Code Archive 2` and the more compressed orientation style of `LUX`.

## Core Territories

### Investigation Operating System

The method layer:

- research workflow
- task decomposition
- synthesis discipline
- model routing
- archive curation

### Agentic Systems

The execution layer:

- custom agents
- agent frameworks
- multi-agent patterns
- session and memory design
- sovereignty and control
- spec-driven execution
- supervisor-worker patterns
- observability and verification

### Tool Intelligence

The evaluation layer:

- IDEs
- automation platforms
- model providers
- local AI tools
- browser and execution tools
- routing and broker layers
- observability and evaluation stacks

### Project Systems

The embodiment layer:

- systems turned into projects
- project architectures
- implementation experiments
- lessons from real builds

### Knowledge Architecture

The synthesis layer:

- semantic compression
- structured insight extraction
- symbolic and conceptual frameworks
- agent-readable knowledge forms

### Strategic Filters

The judgment layer:

- failure modes
- hype resistance
- security awareness
- anti-pattern recognition

## What Makes A Subject Central

A subject is central when it repeatedly affects:

- research method
- project direction
- tool choice
- agent design
- idea generation

## New Durable Pressure From Updates 1.1

The latest update batch makes three things more central than before:

- agent construction as a disciplined practice
- routing and observability as operational infrastructure
- richer project families that now carry more of the research identity

## Usage Rule

Agents should use this map to place a subject before trying to extend it.
If a new idea cannot be placed in the terrain, it is probably either under-formed or misread.
