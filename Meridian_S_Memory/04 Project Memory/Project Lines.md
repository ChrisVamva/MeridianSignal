# Project Lines

## Recurring Project Lines

### Line 1: The Research Operating Environment

Projects that try to turn notes, tools, agents, and workflows into one coherent working environment.

Examples:

- CognitiveEngine
- memory-oriented OpenClaw use
- agent-facing vault summaries

### Line 2: The Publishing And Intelligence Surface

Projects that turn research into an outward-facing product or signal layer.

Examples:

- Orange Tomato
- GitHub Site
- Watchdog structures

### Line 3: The Sovereign Tool Layer

Projects that prioritize local-first tools, personal control, or low-dependency operation.

Examples:

- Scribe
- Doodle Scribe
- Dev's Memory
- Sovereign Voice
- System Monitor

### Line 4: Productivity And Personal Workflow Systems

Projects that turn personal workflow, focus, and command memory into usable software.

Examples:

- FocusFlow
- command and palette systems
- task and focus instrumentation

### Line 5: Automated Evidence Gathering

Projects that move from conceptual research toward structured external signal collection.

Examples:

- Scrapers Project
- tool landscape audits
- market intelligence systems
- Keyword Sleuth

### Line 6: Risk And Adversarial Reading Systems

Projects that interpret dense material against the user's interest rather than passively summarizing it.

Examples:

- Shady Clause Detector

### Line 7: Agentic Execution And Prompt Infrastructure

Projects and systems that formalize how agents plan, route, observe, and execute.

Examples:

- Prompt CLI / Aider / Qwen work
- DSPy planner and verifier architecture
- routing and observability stack design

## Why This Matters

These lines help prevent false novelty.
Many seemingly new ideas are actually extensions of one of these existing trajectories.
