# Update and Compression Rules

## Add To Short Memory Only If It Does One Of These

- preserves a distinction that changes behavior
- improves immediate orientation
- carries a durable operating pattern
- protects against a recurring failure
- keeps current alignment visible

## Do Not Add If It Is

- setup detail
- duplicated explanation
- vendor noise without pattern value
- implementation chatter
- rhetorical emphasis without strategic consequence

## Compression Test

Before a note enters short memory, ask:

1. Can this be understood in under a minute?
2. Does it change how the next session should proceed?
3. Is it preserving a principle rather than clutter?

If not, it belongs elsewhere.
