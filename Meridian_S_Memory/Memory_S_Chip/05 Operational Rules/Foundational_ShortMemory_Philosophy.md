# Foundational ShortMemory Philosophy

## Purpose

Short memory exists to keep the right things near the surface.

It is not a reduced archive.
It is a deliberate priming layer.

## Design Standard

Short memory should help the next session begin with:

- the right strategic frame
- the right operating distinctions
- the right identity alignment
- the right synthesis methods

## What Short Memory Must Preserve

### 1. Near-Surface Truths

These are the truths that should not need rediscovery:

- planning and execution are different jobs
- observability matters
- routing matters
- memory is not archive
- synthesis requires structure

### 2. Current Alignment

Short memory must preserve enough personal and project context to keep the work on-track.

### 3. Behavioral Distinctions

It should preserve the distinctions that prevent waste:

- deep versus shallow work
- durable pattern versus temporary tool
- synthesis versus accumulation

## What Short Memory Must Reject

- archive sprawl
- generic summaries
- tool catalogs without strategic consequence
- long instructions that belong in manuals
- identity overload

## Update Philosophy

Short memory should be updated slowly and deliberately.

A new idea enters short memory only when it proves one of these:

- it keeps recurring
- it changes action
- it protects against a common error
- it clarifies what matters now

## Final Rule

The purpose of short memory is simple:

make the next act of thinking start closer to truth.
