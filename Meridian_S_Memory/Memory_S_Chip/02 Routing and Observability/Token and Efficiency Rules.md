# Token and Efficiency Rules

## Core Rule

Spend expensive reasoning only where it changes the outcome.

## Efficiency Patterns

- use stronger models for blueprinting and semantic decisions
- use lighter models for bounded code translation or repetitive work
- avoid forcing one model to do everything
- keep prompts structurally tight

## Why This Matters

Efficiency is not only about cost.
It preserves system responsiveness, reduces failure pressure, and keeps the stack sustainable.

## Short-Memory Reminder

Token discipline is strategic discipline.
