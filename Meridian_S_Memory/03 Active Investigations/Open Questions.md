# Open Questions

## Core Open Questions

### How should agent memory differ from archive structure?

The system now has enough evidence to know these are not the same thing.
The question is how much compression, prioritization, and frontier logic should be added without losing depth.

### What is the right degree of identity encoding?

Agents need direction, but the research identity is still forming.
The memory should help define values, not prematurely fossilize them.

### How can novelty be made contextual rather than random?

The system should be able to propose useful next ideas that:

- arise from existing themes
- do not merely repeat prior work
- remain meaningfully adjacent to current research

### What belongs in memory versus in the execution stack?

The updates add stronger material on routing, telemetry, verification, and execution contracts.
The open question is which parts should become durable memory and which parts should stay operational source material.

### How should project proliferation be controlled?

The project layer is expanding quickly.
The system needs a sharper distinction between:

- project families
- serious prototypes
- implementation branches
- speculative project ideas

### Which project lines are prototypes and which are long-term pillars?

Some projects are experiments.
Some represent recurring ambitions.
The memory system should help distinguish them.

### How should active context be refreshed over time?

This prototype is human-curated.
Future phases may introduce stronger update logic, but the system needs a clear stable/living distinction first.
