# Idea Frontier

## Purpose

This note tracks where useful novelty can arise from the research without simply repeating what already exists.

## Frontier Rule

A good next idea should satisfy at least two of these:

- grows naturally from an existing research line
- resolves a real tension or gap
- creates a new connection between previously separate areas
- improves the usefulness of the larger system
- avoids duplicating an already explored form

## Current Frontier Zones

### Agent Memory As Research Instrument

The current project itself is a frontier:
not only organizing research, but creating a memory substrate that helps agents think with it.

### Contextual Novelty Systems

There is a promising line around helping agents identify:

- what has already been explored
- what remains unresolved
- where adjacent promising ideas lie

### Spec-Driven Agents With Memory Discipline

There is a live frontier where:

- planner models
- execution models
- memory layers
- verification layers

can be combined without turning the stack into fragile complexity.

### Compression As Interface

Another frontier lies in refining how long-form research is compressed for agents without flattening the meaning.

### Project Cross-Pollination

There is strong potential in connecting:

- project memory
- tool intelligence
- knowledge architecture

so that new product ideas arise from their intersections rather than from isolated brainstorming.

### Routing + Observability As Research Infrastructure

The newest material suggests a productive frontier in making agent systems inspectable by default:

- route by task and model strength
- trace decisions and outputs
- verify outcomes explicitly
- feed lessons back into memory without flooding it

## Current Warning

Do not mistake new packaging for new thought.
If an idea only renames an already existing project line, it is not frontier work yet.
