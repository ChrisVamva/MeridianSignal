# Memory Protocols

## Purpose

This note explains how the memory environment should work.

## Core Distinction

### Stable Files

These change slowly and provide long-term orientation.

Examples:

- research identity sketch
- research terrain map
- project lines
- core research compressions

### Living Files

These change more often and represent the present edge of the system.

Examples:

- active investigations
- open questions
- idea frontier
- repetition boundaries

## Reading Logic For Agents

An agent should usually read:

1. `First Read Packet`
2. `Research Identity Sketch`
3. `Active Investigations`
4. relevant terrain or project files
5. `Idea Frontier`

## Update Logic

At this stage:

- humans decide structural changes
- agents may assist interpretation
- living files may evolve more often than stable files
- new files should only be added when they create a real new function

## Curation Rule

Do not add a note just because a topic exists.
Add it when the note improves orientation, continuity, or frontier reasoning.
