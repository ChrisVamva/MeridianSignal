# Anti-Repetition Logic

## Purpose

This protocol helps the system generate new thought from context rather than recycling its own archive.

## Core Test

Before proposing a new idea, ask:

1. Has this underlying pattern already been explored?
2. Does this proposal resolve an open question or only echo an existing theme?
3. Which project line or terrain zone does it extend?
4. What new mechanism, synthesis, or operational value does it add?

## Acceptable Forms Of Novelty

- a new connection between previously separate subjects
- a better memory or workflow mechanism
- a sharper project form
- a new evaluation lens with real consequences
- a more useful compression format

## Unacceptable Forms Of Novelty

- renaming an existing theme
- repeating enthusiasm without new structure
- proposing a project that duplicates an old ambition without noticing it
- generic “future of agents” speculation with no anchor in the terrain
