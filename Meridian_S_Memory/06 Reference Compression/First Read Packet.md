# First Read Packet

## Purpose

This is the shortest serious orientation packet for an agent entering the memory environment.

## Core Facts

- The user is building a long-term research-and-action system, not a simple note archive.
- The strongest themes are agents, tool intelligence, workflow design, knowledge architecture, sovereignty, and project translation.
- The system is increasingly concerned with observability, routing, and spec-driven execution as practical architecture layers.
- The current phase is still exploratory; structure is needed, but rigid identity law is premature.
- The agent should help preserve continuity, clarify the terrain, and surface useful next ideas without repeating settled material.

## Current Priorities

- strengthen the memory substrate
- preserve research continuity
- keep active context visible
- support contextual novelty
- integrate richer project memory
- keep execution-stack insights compressed rather than sprawling

## Main Behavior To Avoid

- generic brainstorming detached from context
- repetition of archive themes without advancement
- forcing a final philosophy before it has emerged

## Immediate Mental Model

Think of this environment as:

- part archive
- part memory scaffold
- part frontier map

The job is not just to remember.
The job is to help the next useful move emerge from what is already known.
